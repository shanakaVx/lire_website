Domain of Lire

Tone based voice synthersizer framework for voice based application development

Lire = "Read" in French

Features of Lire

Lire is an effective voice synthesizer framework

Lire is based on Sinhala language, which contains the phonetics for Sinhala and Extended Phonetic Alphabet.

Editable speech timeline

Editable text to speech rules and prosodies.

User’s own voice can be fed into the system to generate voice from the text.

API where all the Lire services are exposed

Litreature survey

The research works of Wasala and Gamage (2007) [1] has a set of Sinhala part of speech only to be used in Sinhala Text to Speech Systems. Festival framework used to develop text to speech systems[2] supports different languages, but it is language specific. The proposed system 'Lire' can be used to speak several languages due to the complete set of parts of speech Adoptable dashboard as a special learning tool for Sinhala language. Dashboard is composed of clickable and playable set of tones. [3] The Sinhala IPA developed in the works of Wasala & Gamage (2007) [1] can be extended more inserting foreign vowels and allowing them to add up with Sinhala consonants to produce any tone that can be read by human. A tokenizer identify parts of words and identify parts of words and then convert them to a set of phonetics from the IPA. The Grapheme-to-Phoneme Conversion rules in the work by Wasala et al. (2006) [4] are programmatically embedded in their system and cannot be edited in the program run time. But the new framework is to define a way so that the user can change and add more rules to the framework without re-programming it. Natural Language Processing module does tokenizing and identifies special text representations [5] In the work of Aida-Zade et al. (2013) [7] the NLP module identifies currencies in US dollars, Decimal numbers, ratios, roman numerals, alphanumeric, phone numbers, email addresses, counts and simple mathematical expressions. In the work of Weerasinghe et al. (2007) [8] most features are implemented but still lacks the ability of identifying mathematical expressions, currencies and roman numerals. Lire will have new NLP rules defined to identify these characters The storyboard timeline concept in developing the timeline as in Girgensohn et al. (2000) [8]. Although it is for video editing, it is decided to be used for the speech timeline. As seen in the work of Aida-Zade et al. (2013) [7] the prosodic features are not editable after the speech is generated. But from the new framework full control is given over to the user. The prosodic features has also been discussed in the works of Wasala, Gamage (2007) [1] where Sinhala language’s prosodic features are discussed in depth. In the works of Reshma, Chetna et al (2014) [9] it has been discussed prosodic analysis of speech and implementation of a TTS using Matlab as the technology. But the proposed framework will be using Java Beads library.

Research gap

Tone based voice synthesizer framework that is developed in this project will look through some untouched features that can be added to a tone based framework. Without just being confined to a text to speech system, the proposed approach is to develop a framework which consists of more features. Based on the literature review carried out, we have identified certain areas in the text to speech domain that can be improved. For example, developing an extended IPA, new NLP tokenizing rules, speech editable timeline can be mentioned as the new features in this proposed framework

Objectives

General objectives

To bould a tone based voice synthesizer framework using Sinhala tones, which can be further enhanced to use in other languages as well

To build the framework in a way it is usable for both general users and developer

Specific objectives

Develop a dashboard that contains all the parts of speech, the tones of each individual letter (phonetics and diphones) so that a user can drag and drop the parts of speech into a timeline, make a speech and listen or download it

Design a timeline which enables user to edit the speech and download the speech output in the form of a WAV file. The speech output from the text to speech component is displayed in the fully editable timeline

Develop a text tokenizer with a NLP module to identify text styles like characters, numbers, currencies etc and provide the relevant phonetic equivalent of it. Text tokenizer rules will be customizable by the user and therefore users can add their own rules into the tokenizer so that it could tokenize any other language that the user prefers.

Provide an API for the tone based voice synthesizing framework, API is developed as the main security and the broker for all the internal system calls which are done and as well as develop the API as the gateway to the programmers to access the new framework.
